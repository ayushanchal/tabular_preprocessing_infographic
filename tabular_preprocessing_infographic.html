Tabular Data Preprocessing

Comprehensive Guide by Model Type

Classification Preprocessing
1
Data Cleaning

Key Actions:

  • Remove duplicates using drop_duplicates()
  • Handle missing values with SimpleImputer
  • Detect outliers via Z-scores or IQR
  • Apply domain-specific fixes and type conversions
⚠️ Important: Extreme values can skew decision boundaries in distance-based models like SVM.
2
Feature Engineering

Encoding Strategies:

  • Nominal categories: One-hot encoding (OneHotEncoder)
  • Ordinal variables: Ordinal encoding (OrdinalEncoder)
  • Create polynomial features with PolynomialFeatures
  • Apply domain-driven transforms (log, sqrt, binning)
3
Feature Selection

Selection Methods:

  • VarianceThreshold - Remove constant features
  • SelectKBest(f_classif) - Univariate selection
  • mutual_info_classif - Information-based selection
  • Recursive Feature Elimination (RFE)
  • PCA for dimensionality reduction
4
Normalization/Scaling

Scaling Options:

  • StandardScaler - Zero mean, unit variance
  • MinMaxScaler - Scale to [0,1] range
  • RobustScaler - Robust to outliers
🌲 Tree-based models: Decision Trees, Random Forest, XGBoost do NOT require feature scaling!
5
Data Balancing

Imbalance Techniques:

  • SMOTE - Synthetic minority oversampling
  • RandomOverSampler - Random oversampling
  • class_weight='balanced' - Weighted loss
  • Hybrid methods (SMOTEENN)
6
Model-Specific Tips

Algorithm Requirements:

  • Linear models: Require scaling, avoid multicollinearity
  • Distance-based: k-NN, SVM need scaling
  • Tree-based: No scaling needed, handle categories naturally
  • Naive Bayes: May require non-negative inputs
Regression Preprocessing
1
Data Cleaning

Similar to classification plus:

  • Outliers have larger impact in regression
  • Consider PowerTransformer for skewed targets
  • Use robust regression for extreme outliers
  • Apply Box-Cox transformation for target normalization
📊 Target Variable: Examine and transform skewed target distributions for better modeling.
2
Feature Engineering

Regression-Specific Features:

  • Polynomial features for nonlinear relationships
  • Interaction terms between predictors
  • Domain transforms (log, sqrt) to linearize relationships
  • Feature crosses for complex patterns
3
Feature Selection

Regression Selection Methods:

  • f_regression - Univariate F-tests
  • Pearson correlation analysis
  • Lasso regularization for feature selection
  • Partial Least Squares (PLS) for collinear predictors
  • RFE with regression estimators
4
Scaling & Normalization

Critical for Linear Models:

  • Regularized models (Ridge/Lasso) require scaling
  • Use RobustScaler for outlier-prone data
  • Consider scaling the target for neural networks
  • Tree-based regressors don't need scaling
5
Target Considerations

Target Variable Handling:

  • No class balancing (regression has no classes)
  • Address heteroskedasticity
  • Handle extreme target outliers carefully
  • Consider target transformation for skewed distributions
6
Model-Specific Notes

Algorithm Considerations:

  • Linear regression: Check VIF for multicollinearity
  • SVR & k-NN: Scaling is crucial
  • Tree regressors: Handle mixed types naturally
  • Ensemble methods: Consider native missing value handling
Deep Learning Preprocessing
1
Data Cleaning

Neural Network Requirements:

  • Fully numeric matrix (no NaNs allowed)
  • Convert all data types to float
  • Handle extreme outliers (can destabilize training)
  • Use robust imputation methods
⚡ Critical: Neural networks require completely clean, numeric data with no missing values.
2
Feature Engineering

Deep Learning Approaches:

  • Embedding layers for high-cardinality categories
  • One-hot or ordinal encoding for low-cardinality features
  • Let networks learn interactions automatically
  • Domain transformations still helpful (log, sqrt)
🚀 Pro Tip: Embeddings capture categorical similarities better than one-hot encoding for high-cardinality features.
3
Dimensionality Handling

High-Dimensional Strategies:

  • PCA or autoencoders for dimensionality reduction
  • Dropout for implicit feature selection
  • L1 regularization on input layer
  • Networks can handle many features naturally
4
Scaling (CRUCIAL)

Mandatory for Neural Networks:

  • StandardScaler or MinMaxScaler required
  • Scale inputs to prevent gradient explosion/vanishing
  • Scale continuous targets in regression
  • Use batch normalization layers for internal scaling
🔥 Critical: Unscaled inputs lead to unstable training and poor convergence!
5
Data Balancing

Deep Learning Balancing:

  • Use weighted cross-entropy loss
  • Apply focal loss for extreme imbalance
  • SMOTE and oversampling still applicable
  • Custom loss functions for regression outliers
6
Implementation Notes

Framework Considerations:

  • Use tf.data or DataLoader pipelines
  • Integer-encode categories for embeddings
  • Ensure float32 inputs for optimal performance
  • No invariance to transforms - careful preprocessing essential
